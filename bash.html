1. Introduction to Bash

Bash, or Bourne Again Shell, is one of the most widely used shells in the Linux and Unix operating environments. Not only does it allow you to issue commands interactively, but it also provides a robust scripting language to automate tasks.

This guide is designed for both beginners and intermediate users, helping you write efficient Bash scripts for system administration, automation, and various other purposes.

2. Bash Basics

Before diving into scripts, it’s crucial to get comfortable with some basic Bash commands. These are the core utilities that you’ll use regularly:

  • ls – Lists files and directories in the current directory.
  • cd – Changes your current directory.
  • pwd – Displays the full path of your current directory.
  • mkdir – Creates a new directory.
  • touch – Creates an empty file.
  • rm – Removes files or directories.
  • cp – Copies files or directories.
  • mv – Moves or renames files or directories.
  • cat – Displays the content of a file.
  • echo – Prints text or variables to the terminal.

Let’s go a step further and look at how these commands fit into basic scripts:

Here’s a quick example script that creates a directory, moves into it, creates a file, and displays its path:

#!/bin/bash
mkdir my_directory
cd my_directory
touch new_file.txt
echo "File created at: $(pwd)/new_file.txt"

3. Understanding Shell Variables

In Bash, variables allow you to store data and reuse it throughout your script. Unlike other programming languages, you don’t need to declare the type of the variable. Here’s a simple example:

#!/bin/bash
name="Benjamin"
echo "Hello, $name!"

Notice the $ before the variable name when we use it. This indicates that we’re referring to the value stored in the variable.

3.1 Variable Scope

There are two types of variables in Bash: local and global. Local variables are only accessible within the function where they’re defined, while global variables can be accessed throughout the script.

3.2 Special Variables

Bash provides several special variables that are useful for scripts:

  • $0 – The name of the script.
  • $1, $2, $3, ... – Positional parameters passed to the script.
  • $# – The number of arguments passed to the script.
  • $@ – All the arguments passed to the script.
  • $? – The exit status of the last command.
  • $$ – The process ID of the script.

Here’s a script that uses these special variables:

#!/bin/bash
echo "Script name: $0"
echo "First argument: $1"
echo "Number of arguments: $#"
echo "All arguments: $@"
echo "Process ID: $$"
echo "Exit status of the last command: $?"

4. Conditional Statements in Bash

In Bash, you can control the flow of your script using conditional statements such as if, else, and elif. Here’s the syntax for a simple conditional:

#!/bin/bash
if [ condition ]
then
  # Commands to run if the condition is true
else
  # Commands to run if the condition is false
fi

4.1 Examples of Conditionals

Here’s a practical example where we check if a file exists:

#!/bin/bash
if [ -f "file.txt" ]
then
  echo "File exists."
else
  echo "File does not exist."
fi

4.2 Comparison Operators

To make decisions, you’ll often compare values. Bash supports several comparison operators:

  • -eq – Equal to.
  • -ne – Not equal to.
  • -lt – Less than.
  • -le – Less than or equal to.
  • -gt – Greater than.
  • -ge – Greater than or equal to.

Example of using these operators:

#!/bin/bash
num=10
if [ $num -gt 5 ]
then
  echo "$num is greater than 5"
else
  echo "$num is less than or equal to 5"
fi

5. Loops in Bash

Loops are essential in Bash for performing repetitive tasks. Bash supports for, while, and until loops.

5.1 For Loop

A for loop iterates over a list of items. Here’s an example:

#!/bin/bash
for i in 1 2 3 4 5
do
  echo "Number: $i"
done

5.2 While Loop

A while loop continues as long as a condition is true. Example:

#!/bin/bash
count=1
while [ $count -le 5 ]
do
  echo "Count is $count"
  count=$((count + 1))
done

5.3 Until Loop

An until loop is the opposite of a while loop. It runs until the condition becomes true:

#!/bin/bash
count=1
until [ $count -gt 5 ]
do
  echo "Count is $count"
  count=$((count + 1))
done

6. Functions in Bash

Functions allow you to encapsulate a group of commands and reuse them throughout your script. Here’s the syntax for defining a function:

function_name() {
  # commands
}

6.1 Example of Functions

Here’s an example function that greets a user:

#!/bin/bash
greet() {
  echo "Hello, $1!"
}
greet "Benjamin"

6.2 Functions with Return Values

In Bash, functions don’t return values like in other programming languages. Instead, you can use echo to output a value or modify a global variable. Here’s an example of a function that returns a value:

#!/bin/bash
add() {
  local result=$(( $1 + $2 ))
  echo $result
}
sum=$(add 5 3)
echo "Sum: $sum"

7. Working with Files

Handling files is a key part of Bash scripting. You can create, read, and modify files with simple commands.

7.1 Writing to a File

To write to a file, you can use redirection:

#!/bin/bash
echo "This is a new line" > file.txt

The > operator overwrites the content of the file. To append to a file, use >>:

#!/bin/bash
echo "This line will be added" >> file.txt

7.2 Reading from a File

To read the contents of a file, you can use a loop:

#!/bin/bash
while IFS= read -r line
do
  echo $line
done < file.txt

8. Script Debugging

Debugging is an essential skill in scripting. You can enable debugging in Bash by using the -x option:

bash -x script.sh

This will print each command as it is executed, making it easier to find errors.

You can also add set -x within your script to enable debugging from a certain point:

#!/bin/bash
set -x
echo "This is a debug message"
set +x
echo "Debugging off now"
